Brightcoveは、すべての Brightcove サービスとアプリケーションのステータスを継続的に監視します。サービスに何らかの中断があると、システムステータスページに表示されます。本ページには、予定されているメンテナンス通知もすべて掲載されます。

システム ステータス ページにアクセスするには、 https://status.brightcove.com .

投稿されたすべての通知には、以下の情報が含まれます。

  • インシデントステータス
    • 運用可能 (緑)
    • 通知 (青-システムは現在動作中ですが、遅延または断続的な停止の影響を受ける)
    • 部分的なサービス中断 (黄色)
    • フルサービスの中断 (赤)
      • 注:[システムステータス（SystemStatus）]が[ブルー（Blue）]に変更されても、Brightcoveはユーザに通知を送信しません。ステータスがYellowまたはRedの場合、通知が送信されます。
  • コンポーネント － 影響を受けるサービスまたは領域
  • 場所
    • アジア・太平洋
    • アメリカ
    • エメア
    • 日本
  • 現在の状態とステータス。状態には以下があります。
    • 調査中-エンジニアが問題を調査中
    • 特定-エンジニアが問題を特定し、修正を検討している
    • モニタリング-問題の解決とエンジニアリング監視により、すべてが正常であることを確認する
    • 解決済み-問題が解決しました

投稿されたメッセージはすべて英語で、使用時間は東部標準時 (GMT-5) です。

システム状態の履歴を表示する

システム ステータスの履歴を表示するには、歴史ページ下部のリンク。

システム状態の変更を購読する

サインアップして、システムステータスが変更されたときに通知を受け取ることができます。このセクションの構成は次のとおりです。

  • メールを購読中
  • webhookを購読中
  • サブスクリプションの管理

メールを購読中

電子メールで通知を受け取るには、申し込むボタンにメールアドレスを入力してクリック申し込む.

クリック後添字、あなたの電子メールを確認するように促す以下のメッセージが表示されます。

メールはこちらからno-reply@brightcove.com件名にステータス サブスクリプションがあります。™表示されない場合は、スパムフォルダを確認するか、ITチームに確認してフィルタリングが行われているかどうかを確認してください。

webhookを購読中

ウェブフックは、HTTP POSTで作成された定義されたコールバックです。ステータスの更新がある場合に、コールバックが JSON オブジェクトをポストする URL を指定します。

システム ステータスが変更されたときに通知を受けるようにサインアップするには、申し込むボタン。次に、電子メールアドレスとともに JSON ペイロードを受信する URL を入力します。

システムステータスの変更が送信されると、Webhook アドレスは次のような JSON オブジェクトを受け取ります。

サブスクリプションの管理

登録すると、購読の管理ページへのリンクを記載した電子メールが送られてきます。このページで、お客様の購読について管理できます。通知の受信を希望するコンポーネント/製品/リージョンを選択できます。選択を行ってから、サブスクリプションを保存.

購読を解除するには、登録解除サブスクリプションの管理ページの下部にあるリンク。

Slackで通知を受け取る

Slackには、ステータスの更新を接続するために使用できる独自のWebhookシステムがあります。相談する詳細については、Slack のドキュメントを参照してください。
